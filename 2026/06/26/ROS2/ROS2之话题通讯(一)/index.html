ROS2之话题通讯(一)

ROS2的通讯方式

  • ROS 2 的话题通信（Topic Communication）是最基础、最常用的通信方式，采用发布/订阅（Publish/Subscribe）模型，适用于连续的数据流，例如传感器数据、机器人状态、控制指令等。

本节用到的ros2指令

命令 功能
ros2 run 包名 可执行程序名称 运行指定程序
ros2 node list 查看当前正在运行的节点
ros2 node info 节点名称 查看指定节点信息
ros2 topic echo 话题名称 订阅指定话题，并将接收到的消息打印
ros2 topic info 话题名称 查看某个话题的具体信息
ros2 topic pub 话题名称 消息类型 向指定 Topic 发布消息
ros2 interface show 消息接口 查看指定消息接口的详细定义

终端模拟订阅、发布

1、启动小海龟

  • ros2 run turtlesim turtlesim_node

2、启动启动小海龟控制

  • ros2 run turtlesim turtle_teleop_key
包名 可执行程序名 节点名 备注
turtlesim turtlesim_node /turtlesim 小海龟节点
turtlesim turtle_teleop_key /teleop_turtle 小海龟控制节点

3、查看小海龟节点信息

  • ros2 node info /turtlesim

其中：

  • Subscribers 表示节点订阅的所有话题
    • /turtle1/cmd_vel: geometry_msgs/msg/Twist
      • /turtle1/cmd_vel 订阅的话题名称
        • geometry_msgs/msg/Twist 接收 /turtle1/cmd_vel话题的消息接口
  • Publishers 表示节点发布的所有话题
    • /turtle1/pose: turtlesim/msg/Pose
      • /turtle1/pose 发布的话题名称
        • turtlesim/msg/Pose 发布 /turtle1/pose 话题的消息接口

上述例子中 ：
- /turtle1/cmd_vel话题是小海龟节点接收键盘控制等话题
- /turtle1/pose 话题是小海龟节点发布的位置信息话题

订阅小海龟节点发布的位置话题，查看小海龟的位置信息。

  • ros2 topic echo /turtle1/pose

使用键盘控制小海龟移动，可以看到位置信息也发生变化

再查看一下/turtle1/pose话题的信息，加上-v 可以显示更详细

  • ros2 topic info /turtle1/pose -v
字段 含义 示例 是否常用
Type Topic 的消息类型，格式为 <package>/msg/<Message> turtlesim/msg/Pose ⭐⭐⭐⭐⭐
Publisher count 当前发布该 Topic 的 Publisher 数量 1 ⭐⭐⭐⭐
Node name 发布（或订阅）该 Topic 的节点名称 turtlesim ⭐⭐⭐⭐⭐
Node namespace 节点所在的命名空间（Namespace） / ⭐⭐⭐⭐
Topic type Endpoint 使用的 Topic 消息类型，通常与 Type 一致 turtlesim/msg/Pose ⭐⭐⭐⭐

查看turtlesim/msg/Pose消息接口的详细定义

  • ros2 interface show turtlesim/msg/Pose

可以看到这个和上面我们订阅输出的内容格式是一致的

4、通过发布位置信息控制小海龟移动

上面说到了/turtle1/cmd_vel小海龟通过订阅这个话题，使用geometry_msgs/msg/Twist消息接口获取移动信息的。

  • 查看 geometry_msgs/msg/Twist消息接口的详细定义

小海龟初始状态

其中 linear是线速度，angular是角速度，如果想让小海龟往Y轴正方向移动：

  • ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "{linear : {y : 1.0}}"

    • 默认情况下，它会持续循环发布（约 1 Hz），直到按 Ctrl+C 停止。

可以看到小海龟再持续往前走

如果继续让小海龟往右边走

  • ros2 topic pub --once /turtle1/cmd_vel geometry_msgs/msg/Twist "{linear : {x : 1.0}}"
    • --once 只发送一次
    • -r 后面可以跟频率 -r 10 则1S发10次
